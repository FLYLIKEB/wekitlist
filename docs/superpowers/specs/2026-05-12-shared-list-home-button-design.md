# Shared list 홈 버튼 추가 디자인

## 목적
공유 리스트 화면에서 사용자가 현재 리스트를 벗어나 홈 화면으로 빠르게 돌아갈 수 있도록 우상단에 명확한 이동 버튼을 추가한다.

## 범위
- 대상 화면: `src/components/local/shared-list-page.tsx`
- 대상 위치: 리스트 제목 우측 상단 액션 영역
- 추가 요소: `홈으로` 텍스트 버튼 1개
- 이동 경로: `/`

## 현재 상태
- 전역 레이아웃 `src/app/layout.tsx` 좌상단에는 브랜드형 홈 링크(`Wekitlist`)가 이미 있다.
- 공유 리스트 화면 우상단에는 현재 `새로고침`, `공유` 액션만 있다.
- 공유 리스트 화면에서 홈 이동은 가능하지만, 우상단 액션 묶음에서는 직접 제공되지 않는다.

## 선택한 접근
우상단 액션 영역에 `홈으로` 텍스트 버튼을 추가한다.

### 이유
- 사용자가 요청한 위치와 형태에 가장 직접적으로 맞는다.
- 기존 `공유` 버튼과 동일한 계열의 pill 스타일을 재사용할 수 있어 시각적 일관성이 좋다.
- 아이콘 단독 버튼보다 의미가 즉시 이해된다.

## UI 설계
- 버튼 텍스트: `홈으로`
- 버튼 배치 순서: `홈으로` → `새로고침` → `공유`
- 버튼 스타일: 기존 보조 액션 버튼과 동일한 톤의 rounded pill
- 강조 수준: `공유` 버튼과 같은 계열이되, 화면의 주요 작업 흐름을 방해하지 않는 보조 이동 액션으로 유지

## 동작 설계
- 클릭 시 Next.js `Link` 또는 동등한 클라이언트 내비게이션으로 `/`로 이동한다.
- 새 창이나 외부 이동은 사용하지 않는다.
- 현재 화면 상태를 별도로 저장하거나 확인 모달을 띄우지 않는다.

## 접근성
- 버튼 텍스트 자체가 목적을 설명하므로 추가 아이콘 없이도 의미가 전달되어야 한다.
- 키보드 포커스로 접근 가능해야 한다.
- 기존 액션 버튼과 동일한 hover/focus affordance를 유지한다.

## 구현 메모
- `shared-list-page.tsx` 상단에 필요한 `Link` import를 추가한다.
- 제목 우측 액션 래퍼 내부에 `홈으로` 링크 버튼을 추가한다.
- 기존 우상단 레이아웃 폭을 크게 바꾸지 않는 최소 수정으로 끝낸다.

## 테스트
- 렌더링 테스트: 공유 리스트 화면에 `홈으로` 버튼이 보이는지 확인
- 링크 테스트: 버튼이 `/`를 가리키는지 확인
- 회귀 확인: 기존 `새로고침`, `공유` 버튼이 그대로 노출되는지 확인

## 제외 범위
- 전역 레이아웃의 좌상단 브랜드 링크 변경
- 다른 페이지의 홈 버튼 정렬 통일
- 네비게이션 구조 개편
- 확인 모달, 토스트, 상태 저장 추가
