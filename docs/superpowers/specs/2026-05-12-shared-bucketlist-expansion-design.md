# 공유 버킷리스트 앱 확장 설계

## 개요
기존 Wekitlist MVP를 유지하면서, 실제 함께 쓰는 앱처럼 느껴지도록 공유성, 설치성, 사용감, 항목 표현력을 확장한다.

이번 확장의 목표는 다음 네 가지다.
- 다른 사람에게 바로 보낼 수 있는 공유 링크 경험 추가
- 설치 가능한 PWA 감각 강화
- Apple처럼 차분하고 부드러운 상호작용 추가
- 항목에 선택적 상세정보를 담을 수 있게 하되 기본 입력은 여전히 가볍게 유지

## 제품 원칙
- 기본 사용 흐름은 여전히 1~2개의 입력만으로 끝나야 한다
- 추가 정보는 원하는 사람만 열어 쓰는 방식이어야 한다
- 장식보다 속도와 부드러움이 우선이다
- 삭제와 공유는 자주 쓰는 행동이므로 눈에 잘 띄어야 한다
- 기능은 늘리되 앱의 밀도와 경량감은 유지한다

## 이번 확장 범위
### 포함
- 현재 리스트 URL 복사
- 초대용 짧은 링크 복사
- 복사 완료 팝업/토스트
- PWA 설치 가능 상태 보강
- 설치 유도 프롬프트 노출
- 카드/패널/토스트 수준의 가벼운 애니메이션
- 항목 추가 시 선택적 상세설정 열기
- 상세설정에서 주소 입력
- 상세설정에서 태그 입력
- 각 항목에 생성일 표시
- 해야 할 항목 삭제
- 완료된 항목 삭제
- 즉시 삭제 후 되돌리기

### 제외
- 복잡한 오프라인 동기화
- 푸시 알림
- 이미지 첨부
- 지도 전용 필드
- 항목별 다중 하위 태스크
- 전역 검색
- 드래그 정렬
- 장시간 유지되는 휴지통 화면

## 핵심 경험
### 1. 공유
리스트 상단에는 공유 액션이 있어야 한다. 사용자는 버튼 한 번으로 링크 관련 액션을 열 수 있어야 하며, 여기서 현재 주소 복사와 초대용 짧은 링크 복사를 선택할 수 있어야 한다.

현재 주소 복사는 가장 빠른 공유 수단이다. 초대용 짧은 링크는 전달성이 좋은 보조 수단이다. 두 방식 모두 복사 후 즉시 짧은 피드백이 떠야 한다.

### 2. 설치형 웹앱 감각
앱은 모바일 홈 화면에 추가 가능한 상태여야 한다. manifest, 아이콘, 메타 정보, 설치 유도 흐름이 정리되어야 하며, 설치 가능한 브라우저에서는 자연스럽게 설치 액션을 제안할 수 있어야 한다.

이번 범위에서 목표는 오프라인 우선이 아니라 설치 가능한 앱 같은 인상이다.

### 3. 항목 추가의 2단 구조
기본 항목 추가는 지금처럼 빠르게 제목만 입력하는 흐름을 유지한다. 대신 사용자가 더 자세히 적고 싶을 때만 상세설정을 열어 주소와 태그를 입력할 수 있어야 한다.

즉, 기본 UX는 가볍게 유지하고, 상세 기능은 접혀 있어야 한다.

### 4. 삭제와 되돌리기
사용자는 해야 할 항목과 완료된 항목 모두에서 항목을 삭제할 수 있어야 한다. 삭제는 즉시 반영되어 빠르게 느껴져야 하지만, 실수 방지를 위해 잠깐 동안 되돌리기 기회를 제공해야 한다.

## 정보 구조
### 공유 리스트
기존 `shared_lists`를 유지한다.

추가 활용:
- `invite_token`을 짧은 초대 링크 생성에 그대로 사용

공유 링크 종류:
- 현재 리스트 링크: `/list/{id}`
- 초대 링크: `/invite/{inviteToken}`

### 버킷리스트 항목
기존 항목 모델에 선택적 필드를 추가한다.

필드:
- id
- sharedListId
- title
- linkUrl (nullable)
- tags (nullable)
- createdAt
- completedAt

규칙:
- `title`은 필수
- `linkUrl`은 선택
- `tags`는 선택
- `createdAt`은 자동 저장, 화면에 표시
- `completedAt`은 완료 처리 시 자동 저장
- 삭제는 소프트 삭제가 아니라 실제 삭제를 우선 적용

## 라우팅
### 리스트 라우트
- `/list/{id}`는 현재처럼 메인 리스트 화면

### 초대 라우트
- `/invite/{inviteToken}` 추가
- 토큰으로 리스트를 찾은 뒤 해당 리스트 화면으로 진입시킴
- 사용자는 초대 링크를 눌러도 최종적으로 같은 리스트 경험에 도달해야 함

## 화면 설계
### 1. 리스트 상단 액션 영역
포함 요소:
- 그룹명
- 공유 버튼
- 필요 시 설치 버튼 또는 설치 유도 액션

공유 버튼 동작:
- 작은 팝오버 또는 바텀시트 열림
- 액션 목록:
  - 현재 링크 복사
  - 초대 링크 복사
- 성공 시 짧은 토스트 노출

### 2. 항목 입력 영역
기본 상태:
- 제목 입력 필드
- 추가 버튼
- 상세설정 열기 버튼

상세설정 펼침 상태:
- 주소 입력 필드
- 태그 입력 필드
- 필요 시 태그 안내 문구

원칙:
- 기본 상태는 지금처럼 매우 짧아야 함
- 상세설정은 접었다 펼 수 있어야 함
- 자주 쓰지 않는 기능이 기본 화면을 무겁게 만들면 안 됨

### 3. 해야 할 항목 카드
표시 요소:
- 제목
- 주소 링크(있을 때만)
- 태그 pill(있을 때만)
- 생성일
- 완료 버튼
- 삭제 버튼

상호작용:
- 완료는 한 번 탭
- 삭제는 즉시 반영
- 삭제 직후 하단 토스트에서 되돌리기 제공

### 4. 완료됨 항목 카드
표시 요소:
- 제목
- 주소 링크(있을 때만)
- 태그 pill(있을 때만)
- 생성일
- 삭제 버튼

원칙:
- 완료 영역도 해야 할 항목과 같은 정보 밀도를 유지
- 삭제 방식은 동일하게 적용

## 상호작용 설계
### 복사 피드백
- 복사 성공 시 짧은 팝업 또는 토스트 표시
- 문구는 짧고 명확해야 함
- 예: "링크를 복사했어요"

### 삭제/되돌리기
- 삭제 즉시 카드가 사라짐
- 하단 토스트에 되돌리기 버튼 표시
- 일정 시간 내 되돌리면 복원
- 시간이 지나면 삭제 확정

### 모션
모션은 아래 수준으로 제한한다.
- 상세설정 패널 열고 닫기
- 카드 추가/삭제 시 짧은 전환
- 팝오버/토스트 등장과 퇴장
- 버튼 탭 반응 강화

원칙:
- 짧고 부드럽게
- 툭툭 끊기는 느낌 제거
- 과장된 바운스나 장식용 모션 금지

## 비주얼 방향
- 흰색 기반, 얇은 구분선 중심
- 버튼과 패널 모서리는 부드럽지만 과하게 둥글지 않게 조정 가능
- 태그는 작은 pill 형태로 가볍게 표현
- 토스트와 팝오버는 떠 있는 느낌은 주되 그림자는 과하지 않게
- Apple 느낌은 화려함이 아니라 반응의 매끄러움과 정돈감에서 만들어야 함

## PWA 방향
### 필수
- manifest 정리
- 앱 아이콘 세트 준비
- 메타데이터 정리
- 설치 가능 감지
- 설치 액션 노출

### 이번 범위의 목표
- 사용자가 홈 화면에 추가했을 때 앱처럼 느껴지는 것
- 브라우저 탭 웹사이트가 아니라 설치 가능한 유틸리티처럼 보이게 하는 것

### 이번 범위에서 깊게 하지 않을 것
- 복잡한 캐싱 전략
- 오프라인 작성/동기화 충돌 처리
- 백그라운드 동기화

## 데이터 동작
### 항목 생성
1. 사용자가 제목만 입력하고 생성 가능
2. 상세설정을 열면 주소와 태그를 함께 저장 가능
3. 생성 시 `createdAt` 자동 저장
4. 리스트에는 생성일이 즉시 표시됨

### 항목 삭제
1. 사용자가 삭제 버튼 탭
2. UI에서 즉시 제거
3. 삭제 정보는 잠시 클라이언트에 보관
4. 사용자가 되돌리기 누르면 복원
5. 시간이 지나면 삭제 확정

### 공유 링크 진입
1. 사용자가 `/invite/{token}` 접속
2. 토큰으로 리스트 조회
3. 적절한 리스트 화면으로 이동 또는 렌더링
4. 잘못된 토큰이면 차분한 오류 안내

## 오류 처리
- 복사 실패 시 짧은 실패 메시지 표시
- 잘못된 초대 링크면 진입 실패를 짧게 안내
- 삭제 되돌리기 실패 시 조용한 재시도보다 현재 상태를 우선 정확히 반영
- 주소 형식이 이상하면 저장 전 최소 검증 또는 링크 렌더링 시 방어 필요

## 테스트 우선순위
### E2E
- 리스트 상단에서 현재 링크 복사 가능
- 초대 링크 복사 가능
- 초대 링크 진입 시 같은 리스트 접근 가능
- 상세설정 열기/닫기 가능
- 주소/태그 포함 항목 생성 가능
- 생성일이 화면에 표시됨
- 해야 할 항목 삭제 가능
- 완료된 항목 삭제 가능
- 삭제 후 되돌리기 가능
- 설치 가능 환경에서 PWA 액션이 노출됨

### 단위/통합
- 공유 링크 생성 로직
- 초대 토큰 조회 로직
- 항목 생성 시 optional 필드 처리
- 항목 삭제/복원 로직

## 성공 기준
- 사용자는 링크를 복사해 다른 사람에게 바로 보낼 수 있다
- 사용자는 앱을 홈 화면에 추가할 수 있다
- 사용자는 제목만으로 빠르게 항목을 만들 수 있다
- 필요할 때만 주소와 태그를 넣을 수 있다
- 각 항목에 생성일이 보인다
- 해야 할 항목과 완료 항목 모두 삭제할 수 있다
- 삭제 실수는 되돌리기로 복구할 수 있다
- 전체 상호작용이 지금보다 더 부드럽고 덜 끊겨 보인다
